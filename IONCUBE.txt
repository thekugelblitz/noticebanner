Notice Banner — distribution & IonCube Encoder (v14+)
========================================================

This package is shipped as plain PHP. IonCube is optional: you encode files
yourself with the commercial IonCube PHP Encoder (version 14 or newer) before
reselling.

What to encode
--------------
Encode these PHP entry points (recommended):
  • noticebanner.php
  • hooks.php
  • widget.php
  • license.php           — Contains license validation logic; encode to protect it.

Leave as plain text (do not encode):
  • templates/admin.tpl   — WHMCS loads this as a template; encoding breaks it.

Target PHP version
------------------
Match the PHP version on your customers’ servers (e.g. PHP 8.1 or 8.2 as
required by your WHMCS release). The encoder and loader major versions must
match the runtime PHP.

Customer servers
----------------
They must install the matching IonCube Loader extension. Test after encoding:
module activate, save notice, client-area banner, poll vote, admin widget.

Outbound HTTPS requirement (IMPORTANT)
--------------------------------------
The license engine (license.php) makes an outbound HTTPS POST to (default):

  https://whmcsapi.hostingspell.com/noticebanner-license-api/validate.php

Customer servers MUST be able to reach this URL over HTTPS (port 443).
Optional override: copy noticebanner-license-url.example.php to
noticebanner-license-url.php in the module folder and define NB_LICENSE_API_URL.

Self-signed TLS on the license API
----------------------------------
If the license API presents a self-signed certificate, the module retries the
HTTPS request once with peer verification disabled so validation can succeed.
For production, use a trusted certificate (e.g. Let’s Encrypt) on
whmcsapi.hostingspell.com. Optional: define NB_LICENSE_SSL_VERIFY_PEER as false
in noticebanner-license-url.php (see noticebanner-license-url.example.php).

Cloudflare on the license host
------------------------------
If the hostname is proxied through Cloudflare and returns a “Just a moment…”
HTML page (HTTP 403), server-side PHP cannot complete the JS challenge.
Fix on Cloudflare: add a WAF skip rule for /api/validate.php, or use a
DNS-only (grey-cloud) hostname for the API. See noticebanner-license-api/CLOUDFLARE.txt.

Security note
-------------
Encoded files do not replace secure coding. This build already avoids trusting
client POST for identity on acknowledge, validates poll options server-side,
and restricts webhooks to public http(s) URLs to reduce SSRF risk.
